Extend clover efi bootloader

近段时间使用clover efiboot安装系统，一个是32位cpu＋传统bios＋gpt＋osx snow leopard＋win8 + linux ＋…，一个是64位cpu＋efi bios＋gpt＋ os x marvericks + win8 + linux +…。
没想到clover还是全能型的，linux分区都无需再装grub，就能自动识别和引导。问题是linux的发行版只有64bit有efi引导，32bit下只考虑mbr引导，会带来些麻烦。这个时候注意，无需安装grub，引导的事情交给clover即可。有些发行版本的linux对initrd做了剪裁，udev没有使用part－uuid去绑定分区的设备名，需要重做一下initrd。这点一开始没认识到，又以为是grub在引导，费了不少时间才找到原因。
很多时候，想尝试新的操作系统，一想到要刻光盘或写usb引导盘，有时还要分区，实在麻烦，就放弃了这个念头。要是引导程序能直接引导硬盘上的live cd镜像，对尝鲜最合适不过了。看grub的资料，好像是已经这样做了。没看到clover下引导live cd的介绍，可能是没有。参照grub对iso的引导，做一个clover的扩展，应该不会很费事。
可惜访问sourceforge和github困难，只能慢慢来，先把clover的源码下载来再说。
